Diagrama De Secuencia

Gestión de Usuarios

Descripción

Este módulo facilita a los usuarios registrados realizar préstamos de libros, consultar su historial de préstamos y realizar reservas de libros que no están disponibles actualmente. El sistema asegura la gestión de fechas límite para devoluciones, indicando claramente los plazos establecidos.

También se registra automáticamente el historial de préstamos de cada usuario, incluyendo libros devueltos, en préstamo o reservados. Los bibliotecarios pueden consultar y administrar el estado de los préstamos y reservas. Además, se generan notificaciones automáticas para recordar a los usuarios sobre fechas de devolución próximas o vencidas.

Actores

  • Usuario: Persona que desea realizar un préstamo o reserva de un libro.
  • Sistema de Gestión de Biblioteca (SGB): Sistema encargado de gestionar las solicitudes de préstamos y reservas.
  • Base de Datos de Libros (BDB): Repositorio donde se almacenan los datos de los libros.

Flujo de Mensajes

  • Solicitud de Préstamo o Reserva: El Usuario envía una solicitud al SGB indicando que desea prestar o reservar un libro.
  • Verificación de Disponibilidad: El SGB consulta a la BDB para verificar si el libro solicitado está disponible. La BDB responde al SGB indicando si el libro está disponible o no.

Condiciones

Si el libro está disponible:

  • El SGB registra el préstamo en la BDB.
  • El SGB confirma al Usuario que el préstamo ha sido exitoso.

Si el libro no está disponible:

  • El SGB informa al Usuario que el libro no está disponible para préstamo.

Reserva:

  • El SGB registra la reserva en la BDB.
  • El SGB confirma al Usuario que la reserva ha sido exitosa.
